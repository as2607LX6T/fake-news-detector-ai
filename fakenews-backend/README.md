## Quick Start (5 steps)

### Step 1 — Clone & create virtual environment

```bash
cd fakenews-backend
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### Step 2 — Install dependencies

```bash
pip install -r requirements.txt
```

### Step 3 — Set up environment

```bash
cp .env.example .env
# Edit .env if needed — defaults work out of the box
```

### Step 4 — Generate demo model (first time only)

```bash
python scripts/train_demo_model.py
```

This creates `app/models/model.pkl` and `app/models/vectorizer.pkl`.  
**Replace these with your production-trained files when ready.**

### Step 5 — Start the server

```bash
python main.py
```

Open **http://localhost:8000/docs** — interactive Swagger UI is ready.

## API Reference

### POST /predict

Analyzes a news article or headline.

**Request**
```json
{
  "text": "SHOCKING: Scientists confirm vaccines contain microchips!"
}
```

**Response 200**
```json
{
  "status":     "success",
  "prediction": "Fake",
  "confidence": "93.4%",
  "risk":       "HIGH",
  "word_count": 9
}
```

**Error responses**
```json
{
  "status":  "error",
  "code":    422,
  "message": "Input validation failed.",
  "detail":  "text: Text is too short. Minimum 10 characters required."
}
```

### GET /health/

Liveness probe — returns 200 immediately.

### GET /health/ready

Readiness probe — returns 200 only when model is loaded.

## Using the API from the React Frontend

In your React app (`src/services/api.js`):

```javascript
const response = await fetch("http://localhost:8000/predict/", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ text: articleText }),
});
const data = await response.json();
// data.prediction → "Fake" or "Real"
// data.confidence → "93.4%"
// data.risk       → "HIGH"
```

**CORS** is pre-configured to allow `http://localhost:5173` (Vite dev server).  
For production, set `ALLOWED_ORIGINS=https://your-frontend.vercel.app` in `.env`.

## Using Your Own Trained Model

The API expects two pickle files in `app/models/`:

| File | What it must be |
|---|---|
| `model.pkl` | Any sklearn classifier with `.predict()` + optionally `.predict_proba()` |
| `vectorizer.pkl` | A fitted `TfidfVectorizer` (or compatible transformer) |

**Label convention:** `0 = Fake`, `1 = Real`  
(Edit `predictor.py` line `prediction = "Real" if label_index == 1 else "Fake"` to flip if needed.)

## Deployment

### Render (recommended — free tier available)

1. Push repo to GitHub
2. Go to [render.com](https://render.com) → New → Web Service
3. Connect your GitHub repo
4. Settings:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - **Python version:** set `PYTHON_VERSION = 3.11.9` in Environment
5. Add environment variables:
   - `ALLOWED_ORIGINS` → `https://your-frontend.vercel.app`
   - `RELOAD` → `false`
   - `MODEL_PATH` → `app/models/model.pkl`
   - `VECTORIZER_PATH` → `app/models/vectorizer.pkl`
6. Upload your `.pkl` files (use Render's disk or store in cloud storage + download on startup)
7. Click **Deploy**

### Railway

1. Push repo to GitHub
2. Go to [railway.app](https://railway.app) → New Project → Deploy from GitHub
3. Add the same environment variables as above
4. Railway auto-detects `Procfile` and deploys

### Environment variables for production

| Variable | Production value |
|---|---|
| `RELOAD` | `false` |
| `ALLOWED_ORIGINS` | `https://your-frontend.vercel.app` |
| `LOG_LEVEL` | `WARNING` |
